# 04 Endpoint Concepts

この章では、「エンドポイント（endpoint）」という言葉を  
**できるだけ広い意味から**  整理します。

前章（03_FastAPI_Basics）で FastAPI を実際に動かした経験をもとに、
言葉としての「エンドポイント」を理解することが目的です。

---

## エンドポイントとは何か（広い意味）

エンドポイントとは、

> **外部からアクセスするための「入口」**

を指す言葉です。

- 人がアクセスする入口
- プログラムがアクセスする入口
- ネットワーク越しの入口

このように、「何かを使うための到達点」を  
まとめて **エンドポイント** と呼びます。

---
**エンドポイント** という言葉から、「出口」を連想する人もいるかもしれません。

ここでは、**「端っこ」** というイメージで捉えてもらえると理解しやすくなります。

入口も出口も、どちらも「端っこ」です。
人間にとって分かりやすい「入口」は、コンピュータにとっては  
**外部とつながるための端点（＝エンドポイント）** という位置づけになります。

その先には、実際の処理やサービスがあり、
エンドポイントは、それらを呼び出すための「接続点」になっています。

少し別の言い方をすると、
ドアや門のようなものを想像してもよいかもしれません。

- ドアの向こうにサービスがある  
- ドアの前で「どんな操作をするか（GET / POST など）」を指定する  

というイメージです。

実務では「エンドポイント」という言葉が慣習的に使われているため、
ここでこの言葉とイメージをセットで覚えておきましょう。


---

## Web におけるエンドポイント

Web の世界では、エンドポイントは次のように表現されます。

```text
http://127.0.0.1:8000/
```
これは、

http:// : 通信方法
127.0.0.1 : 接続先
:8000 : ポート番号
/ : パス

を組み合わせた **アクセス先（入口）** です。
---

## FastAPI におけるエンドポイント

FastAPI では、エンドポイントは コードとして定義されます。

```python
@app.get("/")
def read_root():
    return {"message": "Hello World"}
```
ここでは、

- "/" がエンドポイント
- GET メソッドでアクセス可能
- この URL にアクセスすると関数が呼び出される

という関係になっています。
---
## 404 Not Found が意味すること

FastAPI で次のような表示を見たことがあるかもしれません。

```
404 Not Found
```
これは、
指定したエンドポイントが存在しない
という意味です。

- 定義されていない URL にアクセスした
- その入口が用意されていない

という状態を表しています。

## API におけるエンドポイント

API の文脈では、エンドポイントは
- 機能ごとの入口
- データ取得・操作の入口
を意味します。

例：
```
GET /users
POST /users
GET /users/{id}
```

それぞれが 別のエンドポイントです。

## AWS におけるエンドポイント

AWS でも「エンドポイント」という言葉が使われます。
例：
```
https://s3.ap-northeast-1.amazonaws.com
```

これは、

** S3 というサービスへの入口 ** 

リージョンごとに用意された接続先

を表しています。

意味としては、Web や FastAPI と同じく、

サービスにアクセスするための入口

です。

## この章で押さえておきたいこと

エンドポイントとは「入口」を意味する言葉です

Web、API、FastAPI、AWS で使われ方は違いますが、本質は同じです
03 で触った URL や @app.get("/") は、すでにエンドポイントでした
