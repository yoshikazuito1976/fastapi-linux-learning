# util ディレクトリについて

このディレクトリは、  
**各章で共通して利用する設定ファイルや補助的な資材（ユーティリティ）** をまとめるためのものです。

章ごとの学習内容そのものではなく、  
**学習や観察を支えるための共通部品**を配置します。

---

## このディレクトリの役割

- 各章に同じ設定ファイルを重複して置かない
- 章の内容（HTTP / FastAPI の理解）と 実行環境・補助設定（ログなど）を分離する
- 教材としての構造を分かりやすく保つ

この考え方は、実際のアプリケーション開発における  
**「責務の分離」**にもつながります。

---

## 含まれるファイル

### logging.conf

FastAPI アプリを起動する際に使用する  
**共通のログ設定ファイル**です。

- アプリコード（`main.py`）から直接読み込むのではなく  
- **uvicorn 起動時に指定**して使用します

例：

```bash
uvicorn main:app --reload --log-config ../util/logging.conf
```
またはリポジトリルートから起動する場合：
```
uvicorn --app-dir 08_http_methods_overview main:app --reload --log-config util/logging.conf
```
### なぜ util にまとめるのか

- ログ設定は、HTTPメソッドやAPI設計の学習内容そのものではない
- 章ごとに配置すると、設定の重複や差分管理が発生しやすい
- 共通化することで、教材全体の一貫性を保てる

このリポジトリでは、
- 章ディレクトリ = 学ぶ内容
- util ディレクトリ = 学びを支える道具
という役割分担を意識しています。
